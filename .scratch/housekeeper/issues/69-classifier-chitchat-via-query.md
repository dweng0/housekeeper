# Classifier supports chitchat follow-ups via `query` intent

Status: ready-for-agent
Category: bug

## What to build

Update the Intent Classifier so conversational Follow-up Utterances (chitchat) are returned as `type: "query"` with a spoken response, in valid JSON. See ADR 0014.

Today the model frequently replies in natural language to chitchat ("Sounds like a long day‚Ä¶"). The server logs `LLM returned natural language instead of JSON, treating as unknown` and drops the response. The user perceives the dialogue going silent.

Chitchat is folded into the existing `query` type ‚Äî no schema change. The system prompt must permit conversational `query` responses when `conversationHistory` is non-empty, and continue to refuse them when it is empty (Directed Question at session open).

## Acceptance criteria

- [ ] `openai-intent-classifier.ts` system prompt updated to allow conversational `query` responses inside an open conversation, while still requiring JSON output
- [ ] Chitchat Follow-up Utterance ("had a long day", "tell me a joke") returns `{ type: "query", spokenResponse: "...", intentConfidence: number }` ‚Äî never bare prose
- [ ] Directed Question at session open (empty `conversationHistory`) does not invite chitchat ‚Äî current behaviour for non-actionable open turns unchanged
- [ ] Server log `LLM returned natural language instead of JSON` is no longer triggered by routine chitchat (treat any remaining occurrence as a real error)
- [ ] `voice-automation-service.ts` already speaks `intent.spokenResponse ?? intent.response` for `query` ‚Äî no orchestration change required
- [ ] Unit tests in `openai-intent-classifier.test.ts` cover: chitchat follow-up returns valid `query`, low-confidence chitchat returns clarifying path, Directed Question without context does not produce chitchat
- [ ] `src/voice/CONTEXT.md` updated: `Classified Intent` notes that `query` covers chitchat inside an open Conversation Context

## Blocked by

- #68 (Conversation Idle Timeout) ‚Äî bounds the chitchat window so this change cannot keep the mic open forever.
