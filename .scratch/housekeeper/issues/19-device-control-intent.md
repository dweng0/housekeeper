# Device-control intent for immediate commands

## What to build

Add a `device-control` intent type to handle immediate device commands (e.g. "turn on the kitchen light", "switch off the fan") — distinct from `create-automation` which handles rule setup.

Currently, the classifier only recognizes:
- `create-automation` — for setting up triggers/rules
- `query` — for questions
- `set-resident` — for speaker identification

Immediate commands like "housekeeper turn on the lights" get classified as `unknown` because there's no intent type for executing device commands directly.

## Acceptance criteria

- [ ] `ClassifiedIntent` type union includes `device-control`
- [ ] Intent payload includes `deviceLabel` and `command` fields
- [ ] Classifier prompt describes `device-control` intent with examples
- [ ] `VoiceAutomationService.onDirectedQuestion` handles `device-control`:
  - [ ] Resolves device label to topic via `DeviceRepository`
  - [ ] Publishes command to MQTT via `DeviceGateway`
  - [ ] Sends TTS confirmation (or error if device not found)
- [ ] Outcome logged to `LogStore` with new outcome type (e.g. `device-controlled`)
