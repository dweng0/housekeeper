# VoiceAutomationService AEC orchestration

Status: completed
Category: enhancement

## What to build

Extend VoiceAutomationService to orchestrate AEC (acoustic echo cancellation) reference audio broadcast. When TTS is rendered, broadcast audio chunks to all mic-capable devices so they can perform local echo cancellation.

## Acceptance criteria

- [ ] SpeechOutput.speak() callback signature supports chunk streaming: `speak(text: string, onChunk?: (chunk: Buffer) => void): Promise<void>`
- [ ] VoiceAutomationService registers onChunk callback when invoking speak()
- [ ] onChunk callback broadcasts each chunk to:
  - Speaker device (normal tts_stream playback)
  - All mic-capable devices (tts_stream with isAecReference: true flag)
- [ ] Timestamp each broadcast attempt, log chunk arrival and completion time
- [ ] Monitor for backpressure: log if broadcast takes > 50ms per chunk
- [ ] Remove listening window mode switching (onTtsStart/onTtsEnd callbacks that set "stop-word-only" mode) — no longer needed
- [ ] Listening window stays in "normal" mode during TTS (stop-word detection works naturally)
- [ ] Unit tests: verify chunks broadcast to all mic devices, verify isAecReference flag set correctly, verify timing logs

## Blocked by

- #61 VoiceNodeHub TTS lifecycle callbacks (partially — we're removing the mode-switch callbacks but keeping the signals)

## Notes

- Broadcast is async/await with no worker pool (start simple, monitor for bottleneck)
- Document risk: if device count scales, may need worker pool later
- AEC is defense-in-depth: server-side broadcast + client-side local AEC
- Removes ADR 0011 pattern (listening window mode control), replaced by ADR 0012 (AEC)
