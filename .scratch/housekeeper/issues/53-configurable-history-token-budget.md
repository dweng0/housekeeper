# Make history token budget configurable via dashboard

Status: ready-for-agent
Category: enhancement

## What to build

Expose `historyTokenBudget` as a configurable setting in the dashboard, allowing users to adjust the conversation history size limit. Default to 4000 characters. The voice automation service should read this value from config instead of using the hardcoded `HISTORY_TOKEN_BUDGET` constant.

## Acceptance criteria

- [ ] `AppConfig` interface includes optional `historyTokenBudget?: number` field
- [ ] Default value is 4000 (current constant value)
- [ ] Dashboard settings UI shows a numeric input for history token budget
- [ ] Voice automation service reads `config.historyTokenBudget` (falls back to 4000 if not set)
- [ ] Changing budget in dashboard persists to config and takes effect on next utterance
- [ ] Tests pass: config change updates token budget used in classification
