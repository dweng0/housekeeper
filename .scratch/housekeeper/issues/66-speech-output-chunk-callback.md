# SpeechOutput chunk callback for AEC streaming

Status: completed
Category: enhancement

## What to build

Extend SpeechOutput port interface to support chunk callbacks during TTS streaming. Enable VoiceAutomationService to receive audio chunks as they're generated for AEC reference broadcast.

## Acceptance criteria

- [ ] Update SpeechOutput port interface: add optional `onChunk?: (chunk: Buffer) => void` parameter to speak() method
- [ ] Current implementations (Piper TTS adapter) stream chunks and call onChunk for each chunk before send completes
- [ ] Chunk size reasonable (16KB–64KB audio frames, depends on TTS engine)
- [ ] onChunk fires synchronously with generation (no buffer delay beyond TTS engine)
- [ ] Promise<void> still resolves after all chunks have been sent
- [ ] Unit tests: mock callback, verify fires for each chunk, verify order and content
