# Zigbee2MQTT auto-discovery adapter

## What to build

Replace the current raw MQTT `AutoDiscoveryService` implementation with a zigbee2mqtt-aware adapter. Devices are auto-registered into `DeviceRepository` on discovery — no manual topic registration needed.

Mirrors how Home Assistant handles zigbee2mqtt discovery: subscribe to bridge topics, react to device join/leave events, auto-register.

## Port change

Current port is polling-oriented (`getUnregisteredTopics()`). Replace with event-driven callbacks:

```ts
interface AutoDiscoveryService {
  start(): void;
  stop(): void;
  onDeviceDiscovered(handler: (device: Device) => void): void;
  onDeviceRemoved(handler: (topic: string) => void): void;
}
```

## Acceptance criteria

- [ ] `AutoDiscoveryService` port updated to event-driven interface (remove `getUnregisteredTopics`)
- [ ] `Zigbee2MqttDiscoveryService` implements new port
- [ ] Subscribes to `zigbee2mqtt/bridge/devices` — fires `onDeviceDiscovered` for each device on connect
- [ ] Subscribes to `zigbee2mqtt/bridge/event` — fires `onDeviceDiscovered` on `device_joined`, `onDeviceRemoved` on `device_left`
- [ ] Constructs `Device` from zigbee2mqtt metadata: friendly name → `label`, friendly name → `topic`, infers `type` from `exposes` (actuator if has writable features, sensor otherwise)
- [ ] Composition root wires `onDeviceDiscovered` → `DeviceRepository.save`
- [ ] Old raw MQTT `AutoDiscoveryService` impl deleted
- [ ] Dashboard updated to remove "unregistered topics" UI (devices appear automatically)

## Implementation notes

Zigbee2MQTT bridge topics:
- `zigbee2mqtt/bridge/devices` — full device list as JSON array, published on connect
- `zigbee2mqtt/bridge/event` — `{"type": "device_joined"|"device_left", "data": {"friendly_name": "..."}}`

`Device.topic` = zigbee2mqtt friendly name. `Zigbee2MqttGateway` (#21) constructs full topic path from this.

- Zigbee2MQTT device pairing / permit join

### zigbee2mqtt `exposes` type inference

```ts
function inferType(exposes: unknown[]): "actuator" | "sensor" {
  // A device is an actuator if any feature is writable (access flag bit 2)
  function hasWritable(features: unknown[]): boolean {
    return features.some((f: any) => {
      if (typeof f.access === "number" && f.access & 2) return true;
      if (Array.isArray(f.features)) return hasWritable(f.features);
      return false;
    });
  }
  return hasWritable(exposes) ? "actuator" : "sensor";
}
```
