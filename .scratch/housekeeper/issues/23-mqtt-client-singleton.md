# Shared MQTT client instance

## What to build

Create a single shared MQTT client instance injected into both `Zigbee2MqttGateway` (#21) and `Zigbee2MqttDiscoveryService` (#22) via the composition root. Prevents two competing connections to the same broker.

## Acceptance criteria

- [ ] `MqttClient` wrapper created (or raw `mqtt.MqttClient` passed directly) — connects once on startup
- [ ] Broker URL/port sourced from `AppConfig` (not env var hardcoded per-adapter)
- [ ] Single instance injected into both `Zigbee2MqttGateway` and `Zigbee2MqttDiscoveryService`
- [ ] Client lifecycle (connect/disconnect) managed in composition root, not inside adapters

## Implementation notes

Adapters receive the connected client — they do not call `mqtt.connect()` themselves. Composition root owns connection lifecycle.

## Blocked by

None.

## Agent Brief

> *This was generated by AI during triage.*

### Goal

Refactor MQTT connection management so a single `mqtt.MqttClient` instance is created in the composition root and injected into adapters, replacing two competing module-level `mqtt.connect()` calls.

### Current state

Two files each call `mqtt.connect()` independently using the `MQTT_URL` env var:
- `src/mqtt/mqtt-device-gateway.ts` — module-level singleton, exports `reconnectGateway`
- `src/device/auto-discovery-service.ts` — creates client inside `start()`

`AppConfig` (`src/ports.ts`) has no broker URL field. Config is stored in `data/config.json`.

### Steps

1. **Add `mqttBrokerUrl?: string` to `AppConfig`** in `src/ports.ts` and add a default (`"mqtt://localhost:1883"`) in `src/config/json-config-repository.ts`.

2. **Create `src/mqtt/mqtt-client.ts`** — exports a factory:
   ```ts
   import mqtt from "mqtt";
   export function createMqttClient(brokerUrl: string) {
     return mqtt.connect(brokerUrl);
   }
   export type SharedMqttClient = ReturnType<typeof createMqttClient>;
   ```

3. **Refactor `src/mqtt/mqtt-device-gateway.ts`** — remove module-level `mqtt.connect()` and `reconnectGateway` export. Change factory signature to accept `SharedMqttClient`:
   ```ts
   export function createMqttDeviceGateway(client: SharedMqttClient): DeviceGateway { ... }
   ```

4. **Refactor `src/device/auto-discovery-service.ts`** — remove internal `mqtt.connect()`. Accept `SharedMqttClient` as a parameter instead of connecting internally.

5. **Update `src/index.ts`** (composition root):
   - Read `AppConfig` before starting adapters
   - Call `createMqttClient(config.mqttBrokerUrl)` once
   - Pass shared client into both `createMqttDeviceGateway` and `createAutoDiscoveryService`
   - Call `client.end()` on shutdown

### Constraints

- Do not add `Zigbee2MqttGateway` or `Zigbee2MqttDiscoveryService` — those are #21 and #22. Only refactor existing adapters to accept an injected client.
- Keep `DeviceGateway` and `AutoDiscoveryService` port interfaces unchanged.
- `MQTT_URL` env var fallback can be removed — broker URL lives in config now.

### Done when

All existing tests pass. `src/mqtt/mqtt-device-gateway.ts` and `src/device/auto-discovery-service.ts` contain no `mqtt.connect()` calls. Composition root creates exactly one MQTT connection.
