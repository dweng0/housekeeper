# Conversation Context: per-node module

## What to build

New `ConversationContext` class (or module) that maintains the active per-Voice-Node thread of exchanges. Tracks `ConversationTurn[]` (user utterance + system Confirmation Response pairs), enforces a rolling token budget (drop oldest turns first when exceeded), handles timeout expiry (configurable, default 30s from last spoken response), and exposes reset triggers (new Directed Question, resident change).

Does not wire into `VoiceAutomationService` yet — isolated, unit-tested module only.

## Acceptance criteria

- [ ] `ConversationContext` holds a list of `ConversationTurn` (`{ userUtterance: string; systemResponse: string }`)
- [ ] `addTurn(userUtterance, systemResponse)` appends a turn and resets the expiry timer
- [ ] `getHistory(tokenBudget)` returns turns as `Array<{ role: "user" | "assistant"; content: string }>`, dropping oldest first to fit within budget
- [ ] `isOpen()` returns `false` after timeout elapses with no new turn
- [ ] `reset()` clears history and closes the context
- [ ] Timeout duration is injected at construction (default 30s)
- [ ] All above covered by unit tests
