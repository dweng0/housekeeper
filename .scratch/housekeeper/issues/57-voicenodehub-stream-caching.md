# VoiceNodeHub stream chunk caching + replay

Status: ready-for-agent
Category: enhancement

## What to build

Implement stream chunk buffering in `VoiceNodeHub.sendTtsStream()`. Cache buffered chunks with short TTL (~30s) and stream token. Expose retrieval method for replay during interruption flow.

## Acceptance criteria

- [ ] `sendTtsStream()` buffers all chunks as they arrive (AsyncIterable)
- [ ] Generate unique `streamToken` per call (UUID)
- [ ] Store in `Map<string, { chunks: Buffer[], expireAt: number }>`
- [ ] Cache key format: `{nodeId}:{streamToken}`
- [ ] TTL: 30 seconds; auto-cleanup on expiry
- [ ] Add `getStreamBuffer(nodeId: string, token: string): Buffer[] | null` method
- [ ] Returns null if expired or not found
- [ ] Unit tests cover: buffering, expiry, replay retrieval
- [ ] No log spam; quiet operation
