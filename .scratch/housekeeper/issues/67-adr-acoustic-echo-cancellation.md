# ADR 0012: Acoustic Echo Cancellation replaces Listening Window mode control

Status: completed
Category: enhancement

## What to build

Document architectural decision: TTS echo is handled via acoustic echo cancellation (AEC) on mic-capable devices, not via listening window mode gating. Update ADR 0011 or write new ADR 0012 explaining the shift and rationale.

## Acceptance criteria

- [ ] Write or update ADR with standard format
- [ ] Document context: feedback loop issue, mic-only nodes can't gate their input when TTS routes elsewhere
- [ ] Document decision: AEC reference audio broadcast to all mic devices, local cancellation on each device
- [ ] Document rationale: handles echo at acoustic level, scales with device count, not dependent on coordination
- [ ] Document tradeoff: requires Pi nodes to implement WebRTC AEC, adds compute on edge devices
- [ ] Document alternative: listening window mode gating (rejected: doesn't work for devices that can't gate themselves)

**What's this about?**

Document the shift from listening window mode gating to acoustic echo cancellation. Old approach: gate listening window to "stop-word-only" mode during TTS. New approach: send TTS audio to all mic devices as AEC reference, each device does local echo cancellation. Remove listening window mode switching entirely.

**What's already there?**

**How to verify?**

1. Write ADR 0012 (or update 0011):
   - **Status**: Accepted
   - **Context**: Mic-only nodes can't gate their input when TTS routes to different device; listening window mode control only works if both devices controlled together
   - **Decision**: AEC (acoustic echo cancellation) handles echo at acoustic level; server broadcasts TTS audio to all mic devices; each device does local cancellation
   - **Rationale**: 
     - Handles actual acoustic phenomenon (echo) at root cause
     - Scales with device count (no coordination needed)
     - Works for any mic that hears the speaker (not just originating node)
   - **Tradeoff**: Pi nodes must implement WebRTC AEC (CPU cost acceptable for modern Pi), requires timestamp sync between server and devices
   - **Alternatives**: Mode gating (doesn't work for non-originating mics), mic pausing (too aggressive), ignore echo (unacceptable)
